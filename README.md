# Khóa Học Chuyển Ngành AI/ML Engineer (Dành Cho Full-Stack Developer)

Chào mừng các AI assistant tiếp theo! Đây là thư mục lưu trữ tiến trình học tập của học viên chuyển ngành từ Full-stack Web Developer sang AI/ML Engineer (Applied AI). 

Mục tiêu của khóa học là học viên đi qua từng bài, học lý thuyết sau đó lập trình thực hành ngay bài đó để hiểu sâu bản chất (Project-driven learning).

---

## Cấu Trúc Thư Mục Dự Án

* Learning/
  Chứa lộ trình tổng thể và các file tài liệu lý thuyết của từng bài học.
  - Learning/roadmap.md: Lộ trình tổng thể chi tiết 5 giai đoạn trong 4-5 tháng.
  - Learning/bai1.md, Learning/bai2.md...: Các file lý thuyết của từng bài.

* Project/
  Chứa các file code bài tập thực hành viết bằng Python tương ứng với từng bài.
  - Project/bai1.py, Project/bai2.py...: File chứa code ví dụ và bài tập thực hành.

---

## Hướng Dẫn Dành Cho AI Assistant Tiếp Theo

Khi được import vào một IDE mới hoặc máy mới, AI assistant hãy thực hiện các bước sau để hỗ trợ học viên:

1. Đọc lộ trình tổng thể tại Learning/roadmap.md để nắm bắt nội dung chương trình học.
2. Quét thư mục Learning/ và Project/ để xác định bài học lớn nhất hiện tại (Ví dụ: nếu đã có bai1.md và bai1.py, tức là đang học hoặc đã hoàn thành Bài 1).
3. Đọc tệp lý thuyết mới nhất để hiểu bài học gần nhất đang thực hiện.
4. Hỏi học viên: "Bạn muốn học tiếp bài tiếp theo hay muốn thảo luận/chữa bài tập của bài hiện tại?"
5. Khi bắt đầu bài mới (Ví dụ Bài X):
   - Tạo file lý thuyết Learning/baiX.md giải thích rõ lý thuyết cốt lõi của bài học đó.
   - Tạo file code Project/baiX.py chứa các hàm mẫu, code ví dụ và bài tập thực hành kèm bộ kiểm thử tự động nhỏ để học viên chạy lệnh chạy thử.

---

## Quy Tắc Định Dạng Quan Trọng (Style Guide)

* Không sử dụng ký tự double asterisks (dấu hai sao **) để in đậm chữ trong các file markdown (.md) vì học viên thấy ký tự này thô và khó đọc khi xem dạng raw. Hãy dùng viết hoa, thụt lề hoặc trích dẫn khối code thay thế để làm nổi bật thông tin.
